MCMF for Debit/Credit Transaction Fulfillment

How to use Minimum Cost Maximum Flow to determine, in one algorithm run, which transactions pass and which get rejected — respecting arrival order and account balance.

The Problem

InputValue
Account balance₹500
T1 (arrives first)₹200
T2 (arrives second)₹500
T3 (arrives third)₹250
Expected outcomeT1 ✓   T2 ✗   T3 ✓
Key constraint: Ordering must be respected. T2 should be evaluated using the balance after T1 is fulfilled (₹300 remaining). With ₹300 left, T2 (₹500) cannot pass. T3 (₹250) can.

The Critical Modelling Mistake to Avoid

Most implementations make ₹ the unit of flow — i.e. they try to push as many rupees as possible through the network. This causes the algorithm to pick T2 alone (₹500) over T1+T3 (₹450), because 500 > 450.

Correct approach: Make 1 transaction the unit of flow. Balance becomes a capacity constraint. Ordering becomes edge cost.

Why ordering is encoded in cost, not structure

Assigning cost = arrival_rank (T1=1, T2=2, T3=3) means the min-cost objective naturally prefers earlier transactions when there is a tie in flow. Example: if both T1+T2 and T1+T3 were feasible (both giving 2-transaction flow), the algorithm would pick T1+T2 (cost=3) over T1+T3 (cost=4) — matching FIFO order.

Java Implementation

Data model

public class Transaction {
    public final String id;
    public final long   amount;    // in paise to avoid floating point
    public final int    priority;  // arrival order: 1, 2, 3 ...

    public Transaction(String id, long amount, int priority) {
        this.id       = id;
        this.amount   = amount;
        this.priority = priority;
    }
}

Graph builder

public class PaymentFlowGraph {

    // Node indices
    private final int SOURCE;
    private final int BAL;
    private final int SINK;
    private final int[] txnNodes;

    private final MCMFSolver solver;

    public PaymentFlowGraph(List<Transaction> txns, long availableBalance) {
        int n = txns.size();

        // Nodes: 0=SOURCE, 1..n=txn nodes, n+1=BAL, n+2=SINK
        SOURCE    = 0;
        BAL       = n + 1;
        SINK      = n + 2;
        txnNodes  = new int[n];
        for (int i = 0; i < n; i++) txnNodes[i] = i + 1;

        solver = new MCMFSolver(n + 3, SOURCE, SINK);

        for (int i = 0; i < n; i++) {
            Transaction t = txns.get(i);

            // S → Tn: capacity=1, cost=arrival_rank (encodes ordering)
            solver.addEdge(SOURCE, txnNodes[i], 1, t.priority);

            // Tn → BAL: capacity=txn_amount, cost=0 (balance consumption)
            solver.addEdge(txnNodes[i], BAL, (int) t.amount, 0);
        }

        // BAL → SINK: hard balance ceiling
        solver.addEdge(BAL, SINK, (int) availableBalance, 0);
    }

    /**
     * Returns the set of fulfilled transaction IDs.
     * One MCMF run — no loops, no retries.
     */
    public Set<String> solve(List<Transaction> txns) {
        solver.run();  // single invocation of SPFA-based successive shortest paths

        Set<String> fulfilled = new HashSet<>();
        for (int i = 0; i < txns.size(); i++) {
            // Check if the S → Tn edge carries flow (i.e. transaction fulfilled)
            if (solver.flowOnEdge(SOURCE, txnNodes[i]) == 1) {
                fulfilled.add(txns.get(i).id);
            }
        }
        return fulfilled;
    }
}

MCMF Solver (Successive Shortest Paths via SPFA)

public class MCMFSolver {

    private static final int INF = Integer.MAX_VALUE / 2;

    private final int n, source, sink;
    private final List<int[]> graph[];  // graph[u] = list of [v, cap, cost, rev_idx]
    private final Map<Long, Integer> edgeIndex = new HashMap<>();

    @SuppressWarnings("unchecked")
    public MCMFSolver(int n, int source, int sink) {
        this.n = n; this.source = source; this.sink = sink;
        graph = new List[n];
        for (int i = 0; i < n; i++) graph[i] = new ArrayList<>();
    }

    public void addEdge(int u, int v, int cap, int cost) {
        int uIdx = graph[u].size();
        int vIdx = graph[v].size();
        graph[u].add(new int[]{v, cap,  cost, vIdx});
        graph[v].add(new int[]{u, 0,   -cost, uIdx}); // reverse edge
        edgeIndex.put((long) u * n + v, uIdx);
    }

    public int flowOnEdge(int u, int v) {
        int idx = edgeIndex.getOrDefault((long) u * n + v, -1);
        if (idx == -1) return 0;
        int[] fwdEdge = graph[u].get(idx);
        // original cap - remaining cap = flow used
        int[] revEdge = graph[fwdEdge[0]].get(fwdEdge[3]);
        return revEdge[1]; // reverse edge capacity = flow sent on forward edge
    }

    public int[] run() {
        int totalFlow = 0, totalCost = 0;
        while (true) {
            // SPFA (Bellman-Ford queue variant) to find shortest path by cost
            int[] dist = new int[n], prevNode = new int[n], prevEdge = new int[n];
            boolean[] inQueue = new boolean[n];
            Arrays.fill(dist, INF);
            Arrays.fill(prevNode, -1);
            dist[source] = 0;
            Deque<Integer> q = new ArrayDeque<>();
            q.add(source); inQueue[source] = true;

            while (!q.isEmpty()) {
                int u = q.poll(); inQueue[u] = false;
                for (int i = 0; i < graph[u].size(); i++) {
                    int[] e = graph[u].get(i);
                    int v = e[0], cap = e[1], cost = e[2];
                    if (cap > 0 && dist[u] + cost < dist[v]) {
                        dist[v] = dist[u] + cost;
                        prevNode[v] = u; prevEdge[v] = i;
                        if (!inQueue[v]) { q.add(v); inQueue[v] = true; }
                    }
                }
            }

            if (dist[sink] == INF) break; // no augmenting path — done

            // Find bottleneck along the path
            int flow = INF;
            for (int v = sink; v != source; v = prevNode[v])
                flow = Math.min(flow, graph[prevNode[v]].get(prevEdge[v])[1]);

            // Augment
            for (int v = sink; v != source; v = prevNode[v]) {
                int[] fwd = graph[prevNode[v]].get(prevEdge[v]);
                int[] rev = graph[v].get(fwd[3]);
                fwd[1] -= flow; rev[1] += flow;
            }
            totalFlow += flow; totalCost += flow * dist[sink];
        }
        return new int[]{totalFlow, totalCost};
    }
}

Usage

List<Transaction> txns = List.of(
    new Transaction("T1", 200_00L, 1),  // ₹200, arrives first
    new Transaction("T2", 500_00L, 2),  // ₹500, arrives second
    new Transaction("T3", 250_00L, 3)   // ₹250, arrives third
);
long balance = 500_00L; // ₹500 in paise

PaymentFlowGraph graph = new PaymentFlowGraph(txns, balance);
Set<String> fulfilled = graph.solve(txns);

// fulfilled = {"T1", "T3"}
// T2 is absent → rejected

for (Transaction t : txns) {
    String status = fulfilled.contains(t.id) ? "FULFILLED" : "REJECTED";
    System.out.printf("%s (₹%d): %s%n", t.id, t.amount / 100, status);
}
// T1 (₹200): FULFILLED
// T2 (₹500): REJECTED
// T3 (₹250): FULFILLED

Summary

One MCMF run gives you the complete answer:
  • Max flow = max transactions fulfilled within balance
  • Min cost = among all max-flow solutions, prefer earlier-arriving transactions
  • No sequential loops, no retry cycles, no partial state management
